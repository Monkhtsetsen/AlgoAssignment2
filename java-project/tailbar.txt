АЛГОРИТМЫН ЗӨВ БАЙДЛЫН НОТОЛГОО:

GREEDY АЛГОРИТМЫН LOOP INVARIANT:

Өмнөх нөхцөл: Мөр бүр дээр аль болох олон үг байрлана.

Loop Invariant: Алгоритмын давталт бүрийн дараа:
1. Үүссэн мөрүүд бүрэн жигдлэгдсэн байна.
2. Үг бүр яг нэг удаа боловсруулагдсан.
3. Мөр бүрийн урт width-с хэтрэхгүй.

Нотолгоо:
1. Суурь нөхцөл: Эхний үгээс өмнө ямар ч мөр үүсээгүй.
2. Хадгалалт: Давталт бүр дээр:
   - Хэрэв үг нь одоогийн мөрөнд багтахгүй бол шинэ мөр эхэлнэ.
   - Үг бүрийг зөвхөн нэг удаа авч үзнэ.
3. Төгсгөл: Бүх үгс боловсруулагдсан.

Алгоритм зөв ажиллана.


DYNAMIC PROGRAMMING АЛГОРИТМЫН ЗӨВ БАЙДЛЫН НОТОЛГОО:

1. LOOP INVARIANT:

   Өмнөх нөхцөл: minCost[i] нь i-р үгнээс эхлэн
   төгсгөл хүртэлх хамгийн бага зардлыг агуулна.

   Давталтын invariant:
   - minCost[i] = min(cost[i][j-1] + minCost[j]), ∀ j > i
   - breakPoints[i] нь хамгийн бага зардалтай таслах цэг

2. INDUCTION НОТОЛГОО:

   Суурь нөхцөл (Base Case):
   i = n-1 үед: minCost[n-1] = 0 (зөв)

   Индукцийн алхам (Inductive Step):
   i < k ≤ n үед индукцийн таамаглалаар minCost[k] зөв гэж үзье.

   i-д зориулж:
   minCost[i] = min(cost[i][j-1] + minCost[j]), i < j ≤ n

   Энэ нь Bellman-ийн оновчтой зарчим:
   "Дэд асуудлын оновчтой шийдлүүд асуудлын оновчтой
   шийдэлд агуулагдана."

3. ТӨГСГӨЛИЙН НӨХЦӨЛ (Termination):

   Алгоритм i = 0 хүртэл ажиллана.
   minCost[0] нь бүх бичвэрийн хамгийн бага зардал.
   breakPoints массив ашиглан оновчтой таслалтыг олно.

Алгоритм зөв ажиллана.

НАРИЙН ТӨВӨГТЭЙ БАЙДЛЫН ШИНЖИЛГЭЭ:
================================================================================

GREEDY АЛГОРИТМЫН НАРИЙН ТӨВӨГТЭЙ БАЙДАЛ:

Цагийн нарийн төвөг: O(n)
Хаана n = үгийн тоо

Санах ойн нарийн төвөг: O(n)
Хаана n = үгийн тоо

Тайлбар:
- Алгоритм нь үгс дээр нэг л удаа давталт хийдэг.
- Мөр бүр дээр аль болох олон үг оруулдаг.
- Hyphenation-ийн хувьд үг бүрийг шалгадаг.


DP АЛГОРИТМЫН НАРИЙН ТӨВӨГТЭЙ БАЙДАЛ:

Цагийн нарийн төвөг: O(n²)
Хаана n = үгийн тоо

Санах ойн нарийн төвөг: O(n²)
Хаана n = үгийн тоо

Тайлбар:
1. Зардлын матриц бөглөх: O(n²)
2. DP хүснэгт бөглөх: O(n²)
3. Үр дүнг бүтээх: O(n)

Дэд асуудлын тоо: n(n+1)/2 ≈ O(n²)
Дэд асуудлын шийдлийг хадгалах: O(n²)

DP нь Greedy-с илүү санах ой шаарддаг боловч
илүү оновчтой үр дүн өгдөг.
